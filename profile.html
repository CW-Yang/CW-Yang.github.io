個人簡介
我是楊棋崴，1998年出生於桃園市，2020年畢業於宜蘭大學生物機電工程學系，畢業後在台灣圓點 擔任韌體工程師一職，直到今年11月。我喜歡排球、烹飪、音樂，也喜歡寫程式，大學時期以 PLC 及 Arduino 平台為主來撰寫機電整合的控制系統，就職後轉換到以 ARM 架構為基礎的 ST 系列 MCU 繼續撰寫單機的系統控制。
為什麼想成為前端、後端或全端工程師？
由於工作需求而開始接觸物件導向語言 C#，一開始只是寫一些輔助小工具用來做指令測試、除錯和驗證 ，之後到了開發中後期慢慢和團隊裡的軟體前輩們交流、討論專案，他們也會不吝嗇的給予我建議和分享 自身經驗談。相較於韌體和硬體之間的限制及高度相依性，我漸漸覺得軟體要更為活潑有趣，且更具有創造力， 靈感往往可來自於日常生活中的需求，感覺更為親切。
為了成為軟體工程師，曾經做過什麼努力？
一開始為了提升對不同框架語言的熟悉度時常會在下班後在網路上刷題目，或是參考別人寫的有趣的專案作品， 後來有了一定程度的理解後就嘗試著自己寫一些小功能、小遊戲，再後來以軟體工程師為目標後，會去求職網站查看職業的相 關專業能力需求，以此作為學習目標。
如果參與這個計畫，會怎麼安排學習時間？
我於11月底就已離職，所以現階段可將重心與時間投入在訓練上，除了上課時間外，也會另外安排額外的時間來複習，甚至預習 一些基礎觀念，讓自己可以更快進入狀況。
如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？
由於我本身並非資工背景，寫程式的經驗也尚淺，所以可能比較缺乏一些觀念性的概念。但我會按步就班的按照課程進度，從 實作中學習，更有效率的來彌補不足的部分，若有跟不上進度的問題也願意付出更多時間去理解、解決問題。
是否有想要加入的軟體公司？為什麼想加入該公司？
鈦坦科技。
以前就有聽說過該公司的福利待遇還不錯，加上外商背景，更有機會與不同國家的工程師們交流，我也很喜歡它 的團隊精神及理念，且相較於典型的大型企業，自己會比較嚮往更為自由、活潑，甚至更具挑戰性的的工作環境，這也是我選擇成 為軟體工程師的原因之一。
想要對我們說的事情？
我從大學時期就有在關注彭彭老師的 Youtube 頻道，一直都很喜歡老師的教學風格。我是從 Python 及 HTML 開始入門，雖然畢業 以後並非從事軟體相關產業，但有了這些初步理解也讓我在與軟體工程師交流的時候比較能夠進入狀況，學習範圍也不會僅限於系統底 層，能有更廣泛的摸索空間。由於第一屆開始招募時我已經選擇就業而錯過了，所以很高興還能有第二屆的培訓計畫，也希望可以給予 我跟前輩們交流、學習的機會。謝謝！
